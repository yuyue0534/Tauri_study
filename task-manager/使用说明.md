🎯 核心操作
1. 创建任务
- 点击右上角 [新建任务] 按钮
→ 填写任务标题（必填）
→ 选择优先级（紧急/高/中/低/很低）
→ 设置截止日期（可选）
→ 添加标签（可选）
→ 点击 [创建]

2. 任务流转（拖拽操作）
待办 → 进行中 → 已完成

操作方法：
1. 鼠标按住任务卡片
2. 拖动到目标列
3. 松开鼠标

✅ 任务状态自动保存

3. 编辑任务
- 点击任务卡片上的 ✏️ 图标 → 修改信息 → 点击 [更新]

5. 删除任务
- 点击任务卡片上的 🗑️ 图标 → 确认删除

6. 搜索任务
- 顶部搜索框输入关键词
- → 实时显示匹配结果
- → 点击 ❌ 清空搜索

7. 切换视图
- 看板视图：适合拖拽管理，可视化任务流
- 列表视图：适合查看详情，快速浏览

点击顶部 📋 图标切换

⚡ 快捷技巧

优先级管理
- 🔴 紧急 - 立即处理
- 🟠 高 - 今日完成
- 🟡 中 - 本周完成
- 🔵 低 - 有空再做
- ⚪ 很低 - 可延期

标签使用建议
工作、个人、学习、购物、健康...
→ 每个任务可添加多个标签
→ 便于分类和筛选
截止日期提醒
设置截止日期后：
→ 24小时内到期会收到桌面通知
→ 过期任务显示红色边框
📋 典型工作流
方案一：GTD 工作流
- 1. 早上查看 [待办] 列
- 2. 选择 3-5 个重点任务
- 3. 拖到 [进行中]
- 4. 完成后拖到 [已完成]
- 5. 晚上回顾并规划明天

方案二：看板工作流
- 待办：所有新任务
 ↓ 开始工作时拖过来
- 进行中：正在做的任务（建议不超过3个）
 ↓ 完成后拖过来
- 已完成：今日完成任务
⚠️ 注意事项
1. 数据存储
- ✅ 所有数据自动保存到本地
- ✅ 关闭应用不会丢失
- ❌ 暂不支持云同步（未来版本）

2. 备份建议

数据库位置：
- Windows: %APPDATA%\com.taskmanager.app\tasks.db
- macOS: ~/Library/Application Support/com.taskmanager.app/tasks.db
- Linux: ~/.local/share/com.taskmanager.app/tasks.db

复制 tasks.db 文件即可备份
3. 性能优化
- ✅ 任务数 < 1000：流畅运行
- ⚠️ 任务数 > 1000：建议定期归档已完成任务
- 💡 定期删除不需要的任务

4. 通知权限
- 首次使用会请求通知权限
- → 点击 [允许] 启用到期提醒
- → 如拒绝，可在系统设置中开启

💡 使用建议

✅ 推荐做法
- 每天开始工作前规划任务
- 优先级标注准确
- 及时更新任务状态
- 定期清理已完成任务

❌ 避免做法
- 任务堆积在待办列
- 所有任务都标紧急
- 长期不更新状态
- 任务描述过于简单
